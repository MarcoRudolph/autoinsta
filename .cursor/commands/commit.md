# Commit Message Generator

1.  Execute the command `git diff --staged` to see what has changed.
2.  Based on the diff, generate a commit message following **Conventional Commits** format (e.g., `feat:`, `fix:`, `chore:`, `refactor:`).
3.  The message should be concise but descriptive.
4.  Present the message in a code block so I can copy it easily.
5.  Ask if I want you to run the git commit command with this message.
